## ملف الكوكيز (Cookies File)

### الصيغة المدعومة

المشروع يدعم صيغتين لملف الكوكيز:

#### 1. Netscape Cookie File (مُوصى به)

هذه الصيغة تُصدر من إضافة **Cookie-Editor** في المتصفح:

```
# Netscape HTTP Cookie File
# http://curl.haxx.se/rfc/cookie_spec.html
# This file was generated by Cookie-Editor
.magnific.com   TRUE    /       FALSE   1814712527      ab.storage.sessionId...    g%3A459cf...
www.magnific.com        FALSE   /       FALSE   1780238929      Lda_aKUr6BGRr   0
www.magnific.com        FALSE   /       TRUE    1780154328      XSRF-TOKEN      eyJpdiI6IlNOVjl...
#HttpOnly_.magnific.com TRUE    /       TRUE    1780156063      GR_TOKEN        eyJhbGciOiJS...
#HttpOnly_.magnific.com TRUE    /       TRUE    1782744463      GR_REFRESH      AMf-vByO...
www.magnific.com        FALSE   /       FALSE   1780238929      csrftoken       UGjtsm5...
```

**تنسيق الأسطر**: `domain \t flag \t path \t secure \t expiration \t name \t value`

> **مهم**: الأسطر التي تبدأ بـ `#HttpOnly_` تحتوي على كوكيز مهمة (GR_TOKEN, GR_REFRESH). الـ `CookieParser` يقرأها تلقائياً بعد إزالة البادئة `#HttpOnly_`.

#### 2. JSON Cookie File

مصفوفة JSON من كائنات الكوكيز:

```json
[
  {
    "name": "XSRF-TOKEN",
    "value": "eyJpdiI6...",
    "domain": ".magnific.com",
    "path": "/",
    "httpOnly": false,
    "secure": true
  },
  {
    "name": "GR_TOKEN",
    "value": "eyJhbGci...",
    "domain": ".magnific.com",
    "path": "/",
    "httpOnly": true,
    "secure": true
  }
]
```

### الكوكيز المطلوبة (كحد أدنى)

| الكوكي | الغرض | ملاحظة |
|--------|-------|--------|
| `GR_TOKEN` | JWT token للمصادقة مع Firebase/Google | HttpOnly — لا يمكن قراءته من JavaScript. ضروري لجميع الطلبات |
| `XSRF-TOKEN` | CSRF protection token | يتم تحديثه تلقائياً عند كل جلسة. إذا كان منتهياً، يتم تجديده |
| `GR_REFRESH` | Refresh token | HttpOnly. يُستخدم لتحديث GR_TOKEN |
| `magnific_session` | Laravel session cookie | HttpOnly. يحتوي بيانات الجلسة |
| `GRID` | معرف المستخدم الرقمي | يُستخدم في device identification |
| `UID` | معرف المستخدم | يُستخدم في الجلسة |

### الكوكيز التي يتم تخطيها

الكوكيز التالية لا تُحمل لأنها session-bound أو analytics:

| الكوكي | السبب |
|--------|-------|
| `ak_bmsc` | Akamai bot management — مرتبط بالجلسة الحالية |
| `_ga`, `_gid`, `_gat` | Google Analytics |
| `intercom-id`, `intercom-session` | Intercom chat |
| `ph_*`, `posthog` | PostHog analytics |

### كيفية تصدير الكوكيز

1. افتح المتصفح (Chrome/Firefox/Edge)
2. سجّل الدخول إلى https://www.magnific.com
3. ثبّت إضافة **Cookie-Editor** (متوفرة لجميع المتصفحات)
4. افتح Cookie-Editor
5. اضغط **Export** → اختر **Netscape** format
6. احفظ الملف باسم مثل `pinj_magnific.txt`

## متغيرات البيئة (.env)

يمكن استخدام ملف `.env` بدلاً من تمرير الخيارات عبر CLI. الملف النموذجي موجود في `.env.example`:

```env
COOKIES_FILE=cookies.txt
API_HOST=0.0.0.0
API_PORT=8080
BASE_URL=https://www.magnific.com
POLL_INTERVAL=5
POLL_TIMEOUT=180
DOWNLOAD_DIR=./downloads
```

| المتغير | الوصف | الافتراضي |
|---------|-------|----------|
| `COOKIES_FILE` | مسار ملف الكوكيز | — (مطلوب عبر CLI) |
| `API_HOST` | عنوان الربط | `0.0.0.0` |
| `API_PORT` | منفذ السيرفر | `8080` |
| `BASE_URL` | عنوان URL الأساسي | `https://www.magnific.com` |
| `POLL_INTERVAL` | فترة فحص الحالة (ثواني) | `5` |
| `POLL_TIMEOUT` | مهلة التوليد (ثواني) | `180` |
| `DOWNLOAD_DIR` | مجلد التحميلات | `./downloads` |

## الثوابت (constants.py)

### نسب العرض/الارتفاع (AspectRatios)

تُخزن كخريطة `{ratio: (width, height)}` عند الدقة 1k:

```python
DATA = {
    "1:1":  (1024, 1024),
    "16:9": (1344, 768),
    "9:16": (768, 1344),
    "4:3":  (1152, 896),
    "3:4":  (896, 1152),
    "3:2":  (1216, 832),
    "2:3":  (832, 1216),
    "21:9": (1536, 640),
}
```

**طريقة حساب الأبعاد:**
```
الأبعاد النهائية = (width × multiplier, height × multiplier)
multiplier = {1k: 1, 2k: 2, 4k: 4}[resolution]
```

مثال: `16:9` بدقة `2k` = (1344×2, 768×2) = (2688, 1536)

### الدقات (Resolutions)

| النوع | القيم |
|-------|-------|
| صور | `["1k", "2k", "4k"]` |
| فيديو | `["1080p", "720p", "480p"]` |

### فئات الصور (ImageCategories)

| الثابت | القيمة |
|--------|-------|
| `CHARACTER` | `"character"` |
| `PRODUCT` | `"product"` |
| `IMAGE` | `"image"` |
| `COMPOSITION` | `"composition"` |
| `STYLE` | `"style"` |

### حدود المراجع (ReferenceLimits)

| الثابت | القيمة | الوصف |
|--------|--------|-------|
| `MAX_IMAGE_REFS` | 14 | أقصى عدد مراجع للصور |
| `MAX_VIDEO_IMAGE_REFS` | 9 | أقصى عدد صور مرجعية للفيديو |
| `MAX_VIDEO_VIDEO_REFS` | 2 | أقصى عدد فيديوهات مرجعية |
| `MAX_VIDEO_AUDIO_REFS` | 2 | أقصى عدد ملفات صوت مرجعية |

| الموقع | BASE_URL | API_PREFIX |
|--------|----------|-------------|
| magnific.com | `https://www.magnific.com` | `/app` |

## User-Agent (user_agent.py)

| الثابت | القيمة |
|--------|-------|
| `IMPERSONATE` | `chrome136` — يُستخدم مع curl_cffi |
| `DEFAULT` | Chrome 136 على Windows 10 (User-Agent كامل) |
| `CHROME_136_WINDOWS` | نفس DEFAULT |

### لماذا Chrome 136؟

- يوفر **15 فتحة جهاز** (device slots) على منصة Magnific
- بصمة TLS حديثة لا تُحظر من Cloudflare
- مدعوم في curl_cffi (أحدث إصدار متاح)

## Rate Limiting (محلي)

إعدادات Rate Limiter في السيرفر المحلي:

| الإعداد | القيمة الافتراضية | الوصف |
|---------|------------------|-------|
| `max_requests` | 20 (عند تشغيل serve) | أقصى عدد طلبات |
| `window` | 60 ثانية | نافذة الزمن |
| `exempt_paths` | `/api/health`, `/docs`, `/openapi.json`, `/redoc` | مسارات مستثناة |

المسارات التالية مستثناة من Rate Limit:
- `/api/health` — فحص الحالة
- `/docs` — Swagger UI
- `/openapi.json` — مواصفات OpenAPI
- `/redoc` — ReDoc

## Logging

نظام السجلات يُخرج إلى stdout بالصيغة:

```
2026-05-30 21:48:35 | INFO     | magnific | Discovered 6 image models, 9 video models
```

| الحقل | الوصف |
|-------|-------|
| التاريخ/الوقت | `YYYY-MM-DD HH:MM:SS` |
| المستوى | DEBUG, INFO, WARNING, ERROR |
| الوحدة | اسم الـ logger |
| الرسالة | محتوى السجل |

مستويات السجلات (من الأدنى للأعلى):
- `DEBUG` — تفاصيل طلبات HTTP (URLs, headers)
- `INFO` — تدفق العمل الأساسي (تم التوليد، تم الحفظ)
- `WARNING` — مشاكل غير حرجة (Rate limit, retry)
- `ERROR` — فشل العملية (HTTP errors, exceptions)
